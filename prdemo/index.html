新增设计笔记

重点
标题：用户教育

为什么这样设计：先解释“技能”是什么，再让用户进入创建，避免一上来就让用户面对复杂配置。

一条链接承载三层内容

Demo 展示方案，批注解释判断，云端空间同步讨论。三层叠在一起，构成一份可阅读、可体验、可持续更新的产品交付物。

  • 第一层：可交互 Demo 让读者直接点击和查看产品方案，减少纯文字描述带来的理解偏差。
  • 第二层：产品批注层 序号、标题、详情卡片和编辑入口直接贴在页面元素旁边，不污染 Demo 源码。
  • 第三层：云端协作空间 分享链接携带空间 ID，同一个链接下的批注来自同一个事实来源。

为什么需要：AI 时代，静态 PRD 越来越跟不上 Demo

AI 可以快速生成可交互 Demo，产品沟通的重心正在从“读文档”转向“看见和体验方案”。但 Demo 本身承载不了全部产品思考，PRDemo 解决的正是这层缺口。

问题一

文档是静态的，Demo 是动态的

传统 PRD 需要读者在脑中还原交互流程；Demo 可以直接被点击、查看和验证。沟通对象越多，动态体验带来的效率差异越明显。

问题二

文档很难跟上讨论和迭代

产品想法会随着评审持续变化，静态文档很容易滞后。把逻辑挂在 Demo 元素旁边，可以让描述和页面尽量保持一致。

思考难以只靠 Demo 承载 入口为什么放这里、用户为什么会点、风险在哪里、后续怎么验证，这些逻辑需要贴着界面留下来。
静态 HTML 缺少协作能力 传统 Demo 通常只是一个可查看页面，PRDemo 通过云端批注空间让它具备评论、管理和持续更新能力。
版本容易漂移 Demo 一版、文档一版、评论又一版，最终很容易对不上。统一链接和空间能降低维护成本。

如何解决：围绕 Demo 生产、解释、同步和协作

PRDemo 把“生成 Demo、挂载思考、同步迭代、协作评审”串成一条线。产品经理不用等到 Demo 做完再补文档，产品逻辑会在创作过程中自然沉淀。

01

AI 生成 Demo

产品经理先用 AI 或代码快速做出 HTML Demo，让想法变成可以被讨论、被点击、被验证的界面。

02

挂载产品逻辑

hover 到区块旁边出现 +，或者 Alt + 点击，直接补充“为什么这样设计”。批注发生在思考现场。

03

看板管理批注

批注多了以后，通过看板搜索、定位、改标题、改状态色卡，像管理一份可交互 PRD 一样管理 Demo。

04

分享链接协作

链接里带着 `prd_space`。别人安装插件后打开同一链接，可以查看同一份批注，也可以继续补充评论。

批注里写什么：产品判断，而不是零散评论

每条批注都围绕一个页面区块展开，承载设计原因、用户问题、预期行为和验证点。读者可以顺着界面理解产品取舍。

批注详情：用户教育

为什么这样设计

新用户不知道“技能”是可复用的任务能力，先用示例和推荐入口降低概念门槛。

解决什么问题

避免用户直接进入复杂配置页后放弃，先建立“我能用它解决什么问题”的认知。

预期用户行为

用户先浏览推荐技能，再选择一个模板进入创建，降低从零填写表单的压力。

后续验证点

观察创建入口点击率、模板使用率，以及用户是否能在 1 分钟内完成首次技能创建。

产品亮点

PRDemo 的目标不是替代所有产品文档，而是让 Demo 成为更高效的沟通中心：界面、逻辑、讨论和迭代都围绕同一个链接发生。

+

不侵入 Demo

作为浮层套在任意 HTML 页面外面，不要求业务 Demo 接入 SDK，也不改变 Demo 本身结构。

#

元素级产品逻辑

每条批注绑定页面选择器、标题、内容、时间和状态色卡，让读者快速知道这条判断属于哪个功能区块。

☁

云端唯一来源

同一个 `prd_space` 下的批注保存在云端，确保分享出去的 Demo 和产品说明指向同一份实时来源。

▤

批注看板

集中搜索、定位、编辑、复制和删除批注，解决批注多了以后分散难管理的问题。

⌥

不中断交互

编辑模式保留 Demo 原本点击行为，可以先打开菜单、弹窗，再对出现的元素追加批注。

◎

轻量阅读体验

默认只显示序号和短标题，hover 或选中后展开详情，降低批注对页面完整度的干扰。

最终交付：一个可协作的 Demo + PRD

PRDemo 的出发点是一种新的 PM 生产方式：用 AI 快速生成可交互 Demo，再把产品背后的分析、逻辑和讨论挂载在 Demo 上。最终交付物从一份静态 PRD，变成一个可以体验、可以阅读、可以协作更新的产品空间。

  • 范式判断：AI 降低了原型生产成本，产品沟通可以从静态 PRD 走向动态 Demo。
  • 核心矛盾：Demo 能展示交互，但无法独立承载完整的产品逻辑、假设和取舍。
  • 设计方案：以 Chrome 插件承载批注层，让产品思考挂载在任意 HTML Demo 上。
  • 协作演进：从本地批注文件迭代到 Supabase 云端空间，让 Demo + PRD 具备协作属性。
